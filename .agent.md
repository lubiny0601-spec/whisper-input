# 项目协作规则

## 测试与验证

- 所有前端视觉、布局、样式改动，完成后必须通过真实浏览器或应用截图验证关键页面；截图无明显错位、重叠、截断后，才能在总结中确认完成。
- 修复非热键相关功能时，不得弱化既有热键回归用例。尤其 Windows RightAlt/AltGraph 同时涉及前端录制器和后端低层键盘 hook：设置页录制快捷键期间，后端 hook 必须放行当前触发键，避免把 RightAlt 吞掉导致无法重新录制或触发。

## 本地目录管理

- Whisper Input 相关工作只在 `C:\Vibe Coding Project\Whisper-input` 下进行。
- 本地长期只保留两个目录：`source` 作为最新源码目录，`app` 作为最新应用程序/安装包留存目录。
- 发布新版本后，把临时构建、打包缓存、旧版本输出和多余项目副本清理掉；不要按版本号创建新的源码副本。
